# ADR-CAT-001A — Primer Tiempo Lógico y Apertura de Historia Clínica PyME

## Contexto

El primer contacto entre SmartPyme y una PyME no debe entenderse como un onboarding conversacional ni como un chat aislado.

Debe entenderse como un:

```text
primer tiempo lógico clínico-operacional.
```

Este tiempo lógico constituye:

- la apertura del caso;
- la habilitación clínica del tenant;
- el inicio de la memoria operacional;
- el comienzo del laboratorio PyME.

El objetivo no es responder mensajes.

El objetivo es:

```text
transformar caos operativo difuso
→ en estructura operacional contrastable.
```

---

# Definición

El primer tiempo lógico es el ciclo completo mediante el cual SmartPyme:

```text
recibe una demanda,
realiza anamnesis,
formula hipótesis,
pide evidencia,
contrasta documentación,
genera hallazgos,
y emite el primer informe clínico-operacional.
```

El ciclo termina cuando:

```text
el sistema entrega un primer informe consistente
+
el dueño reconoce valor o claridad.
```

Ese cierre constituye:

```text
la apertura formal de la historia clínica PyME.
```

---

# El chat no es el producto

El chat es solamente:

```text
la interfaz de recepción.
```

El producto real es:

```text
la recepción clínica-operacional de la PyME.
```

La conversación es parte constitutiva del laboratorio.

---

# Orden diagnóstico correcto

```text
1. Recepción
2. Taxonomía inicial
3. Anamnesis conversacional
4. Hipótesis iniciales
5. Pedido documental
6. Contraste documental
7. Laboratorio inicial
8. Primer informe
9. Apertura de historia clínica PyME
```

No debe intentarse diagnosticar antes de:

```text
- entender el tipo de PyME;
- comprender el dolor;
- recibir evidencia suficiente.
```

---

# Taxonomía inicial

La primera capa estructural es la taxonomía.

El sistema necesita identificar:

```text
qué tipo de organismo operacional tiene delante.
```

Ejemplos:

```text
- rubro;
- comercial / industrial / servicio;
- cantidad de empleados;
- produce o revende;
- existencia de stock;
- canales de venta;
- herramientas actuales.
```

---

# Patologías PyME

Luego de la taxonomía, SmartPyme debe contrastar síntomas contra patologías frecuentes.

Ejemplos:

```text
- margen erosionado;
- costos desactualizados;
- stock fantasma;
- caja mezclada;
- conciliación inexistente;
- dependencia extrema del dueño.
```

En esta fase el sistema no diagnostica.

Solo:

```text
abre hipótesis.
```

---

# Conversación no invasiva

La anamnesis debe ser:

```text
- progresiva;
- pausable;
- retomable;
- explicativa;
- no intrusiva.
```

La PyME no debe sentir:

```text
“me están haciendo llenar un ERP.”
```

Debe sentir:

```text
“me están ayudando a entender qué me pasa.”
```

---

# Pedido documental contextual

SmartPyme no debe pedir archivos genéricamente.

Debe relacionar:

```text
síntoma
↔
evidencia necesaria.
```

Ejemplo:

```text
“No sé si estoy ganando plata.”
```

requiere:

```text
- ventas;
- costos o facturas;
- lista de precios;
- extracto o caja.
```

---

# Relación con BEM

BEM no participa en la conversación clínica.

BEM participa en:

```text
la reducción de entropía documental.
```

Pipeline:

```text
archivo real
→ BEM
→ evidencia curada
→ contraste SmartPyme
→ hallazgos.
```

---

# Primer informe

El primer informe es:

```text
el primer documento constitutivo del caso.
```

Debe contener:

```text
- síntomas detectados;
- hipótesis principales;
- evidencia recibida;
- evidencia faltante;
- hallazgos iniciales;
- riesgos visibles;
- próximos pasos.
```

Incluso un informe parcial puede ser válido si es:

```text
- consistente;
- honesto;
- útil;
- trazable.
```

---

# Apertura de Historia Clínica PyME

Cuando el primer informe es emitido:

```text
el tenant deja de ser anónimo.
```

A partir de ese momento existe:

```text
- historia operacional;
- síntomas históricos;
- evidencia histórica;
- taxonomía viva;
- evolución futura.
```

La memoria comienza cuando el sistema registra estructuradamente el primer tiempo lógico.

---

# Persistencia obligatoria

El sistema debe persistir:

```text
- tenant_id;
- frases textuales;
- anamnesis originaria;
- taxonomía inicial;
- hipótesis iniciales;
- patologías sospechadas;
- documentos pedidos;
- documentos recibidos;
- evidencia curada;
- hallazgos;
- informes emitidos.
```

Persistir solamente mensajes de chat es insuficiente.

---

# Definición final

Hermes no es un chatbot empresarial.

Hermes es:

```text
una recepción clínica-operacional PyME.
```

La conversación no precede al laboratorio.

La conversación:

```text
es el comienzo del laboratorio.
```
