# SmartPyme — Método hipotético-deductivo aplicado a PyMEs

## Estado
Principio metodológico rector.

## Principio central

SmartPyme no arranca afirmando un problema.
SmartPyme arranca formulando una hipótesis verificable.

El dolor del dueño no es diagnóstico.
El síntoma no es patología confirmada.
La patología posible no es hallazgo.
El hallazgo requiere evidencia, comparación y diferencia cuantificada.

## Secuencia metodológica

1. El dueño expresa un dolor.
2. El sistema traduce ese dolor a síntoma operativo.
3. El sistema formula una hipótesis investigable.
4. El sistema deduce qué variables y evidencia hacen falta.
5. El sistema solicita o recupera esa evidencia.
6. El sistema cura los datos.
7. El sistema valida condiciones.
8. El sistema compara evidencia contra fórmulas, referencias o catálogos.
9. El sistema confirma, descarta o declara evidencia insuficiente.
10. El sistema produce reporte diagnóstico.
11. El sistema propone acción.
12. El dueño decide.

## Ejemplo

Dolor:
“Creo que estoy perdiendo plata.”

Síntoma operativo:
Sospecha de pérdida de margen.

Hipótesis investigable:
“Investigar si existe pérdida de margen por desalineación entre costos reales y precios de venta durante un período determinado.”

Deducción:
Para verificar esta hipótesis se necesitan:
- ventas reales
- costos reales o facturas de proveedor
- período
- productos o familias
- descuentos/promociones si aplican

Prueba:
Comparar precio real cobrado contra costo real o costo de reposición.

Resultado posible:
- CONFIRMED
- NOT_CONFIRMED
- INSUFFICIENT_EVIDENCE

## Regla de evidencia

Sin evidencia trazable no hay diagnóstico confirmado.

## Regla de comparación

Sin comparación entre fuentes no hay hallazgo fuerte.

## Regla de cuantificación

Sin diferencia cuantificada no hay hallazgo accionable.
