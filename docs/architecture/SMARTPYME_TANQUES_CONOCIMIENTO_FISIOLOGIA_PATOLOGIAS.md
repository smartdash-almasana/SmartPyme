# SmartPyme — Tanques de conocimiento, fisiología PyME y patologías operativas

## Estado

Documento conceptual y arquitectónico.

Este documento fija una capa central del producto SmartPyme: los **tanques de conocimiento** como base transversal para interpretar, diagnosticar, repreguntar, educar y transformar la concepción del negocio en la cabeza del dueño PyME.

No implementa código.
No modifica Supabase.
No crea endpoints.
No reemplaza el flujo P0 actual.

Define el puente entre:

```text
anamnesis conversacional
+ ingesta documental
+ fórmulas de administración de empresas
+ patologías PyME
+ buenas prácticas
+ hallazgos accionables
+ transformación mental del dueño
```

---

## 1. Principio rector

SmartPyme no debe limitarse a leer archivos ni a producir reportes técnicos.

El producto real es provocar una transformación:

```text
Antes: “tengo un caos y no entiendo qué pasa.”
Después: “ahora entiendo mi negocio, veo qué choca contra qué y sé dónde investigar o actuar.”
```

El objetivo no es solamente entregar datos.

El objetivo es cambiar la concepción del negocio en la cabeza del dueño.

---

## 2. La PyME como organismo vivo

La analogía estructural correcta es:

```text
La PyME = organismo vivo.
```

Sus áreas no son módulos aislados. Son órganos, sistemas y flujos vitales:

```text
ventas
stock
compras
caja
margen
clientes
proveedores
producción
administración
logística
comercialización
tiempo
precios
costos
```

Si una variable se enferma, puede desestabilizar el organismo entero.

Las enfermedades PyME pueden empezar en forma silenciosa:

```text
margen que se erosiona
stock que se descuadra
precios que quedan atrasados
caja que no cierra
tiempo operativo que se pierde
compras mal negociadas
productos muertos
sobrestock
substock
fuga administrativa
```

A veces el dueño naturaliza el síntoma hasta que el desequilibrio se vuelve crítico.

SmartPyme debe funcionar como una medicina operacional para PyMEs.

---

## 3. La clínica-operacional

La entrevista inicial no es onboarding.

Es anamnesis empresarial.

En medicina, el médico no empieza mirando análisis. Empieza escuchando al paciente.

En SmartPyme:

```text
el dueño relata
el chatbot escucha
el sistema pregunta
se registran síntomas
se formulan hipótesis
se piden estudios
se confronta evidencia
se detecta patología
se emite diagnóstico operacional
```

El chatbot/Hermes cumple el rol de médico entrevistador o copiloto clínico-operacional.

Los documentos cumplen el rol de estudios de laboratorio.

BEM.ai cumple el rol de laboratorio documental: lava, cura, estructura y normaliza archivos caóticos.

SmartPyme cumple el rol de sistema diagnóstico: interpreta, contrasta, calcula, pregunta y reporta.

---

## 4. La capa educativa

SmartPyme no es solo clínica.

También tiene una capa educativa.

Muchas veces el dueño no quiere implementar inmediatamente. Primero quiere saber:

```text
qué le pasa a su PyME
qué no está viendo
qué choca contra qué
dónde se pierde dinero
dónde se pierde tiempo
por qué vende pero no queda plata
por qué trabaja mucho pero no mejora
por qué toma medidas impulsivas que empeoran el problema
```

La salida de SmartPyme debe producir comprensión.

No alcanza con decir:

```text
“hay 22 filas duplicadas”
```

Debe explicar:

```text
qué significa eso para la economía del negocio,
qué órgano afecta,
qué patología sugiere,
qué evidencia falta,
y qué debería mirar el dueño.
```

## 7. Fórmulas fundamentales

El sistema debe incorporar fórmulas y criterios que permitan investigar la economía real de una PyME.

Áreas mínimas:

### 7.1 Costos

Una PyME puede vender y aun así perder dinero si no entiende sus costos.

Conceptos mínimos:

```text
costo directo
costo indirecto
costo fijo
costo variable
costo unitario
costo de reposición
costo real vs costo registrado
margen bruto
margen de contribución
punto de equilibrio
```

### 7.2 Precios

Precio es entidad crítica.

No es solo un número.

Un precio puede estar atrasado, mal calculado, mal actualizado, mal trasladado o desconectado del costo real.

Conceptos mínimos:

```text
precio de venta
precio de lista
precio efectivo
precio con descuento
precio de reposición
precio histórico
precio relativo
traslado de aumento
brecha costo-precio
```

### 7.3 Tiempo

Tiempo es variable económica.

La PyME pierde dinero también por tiempo mal usado.

Conceptos mínimos:

## 8. El catálogo de patologías como motor diagnóstico

Cada caso que llega a SmartPyme debe confrontarse con el catálogo de patologías.

Flujo:

```text
relato del dueño
→ síntomas detectados
→ patologías posibles
→ hipótesis diagnósticas
→ evidencia requerida
→ preguntas al dueño
→ archivos solicitados
→ cálculo
→ hallazgo
```

Ejemplo:

Relato:

```text
“vendo mucho pero nunca tengo plata”
```

Patologías posibles:

```text
erosión de margen
flujo de caja desordenado
precios atrasados
financiación tóxica
stock inmovilizado
compras mal negociadas
```

Evidencia requerida:

```text
ventas
compras
costos
precios históricos
stock
cuentas corrientes
caja/banco
```

Preguntas:

```text
¿Qué período querés revisar?
¿Tenés lista de precios histórica?
¿Los precios se actualizan por costo de reposición?
¿Vendés más a contado o financiado?
¿Tenés stock inmovilizado?
```

## 9. El catálogo fisiológico como referencia de normalidad

La investigación no solo busca enfermedad.

También compara contra funcionamiento sano.

Ejemplos:

```text
Una empresa sana debería conocer su margen por familia.
Una empresa sana debería saber qué productos inmovilizan capital.
Una empresa sana debería distinguir caja de ganancia.
Una empresa sana debería medir rotación.
Una empresa sana debería saber cuánto tiempo tarda en recuperar dinero.
Una empresa sana debería actualizar precios según reposición, no solo intuición.
```

La fisiología PyME permite decir:

```text
esto no está necesariamente enfermo, pero está ciego
esto funciona, pero sin control
esto vende, pero no sabe qué gana
esto produce, pero no sabe dónde pierde tiempo
```

## 10. Tres cerebros de SmartPyme

### 10.1 Cerebro conversacional

Escucha, pregunta, interpreta, detecta dolor y recopila contexto.

Responsable:

```text
Hermes/chatbot
```

### 10.2 Cerebro clínico

Detecta patologías, formula hipótesis, pide evidencia y contrasta.

Responsable:

```text
catálogo patológico + motor de hipótesis
```

### 10.3 Cerebro fisiológico/educativo

Sabe cómo debería funcionar una PyME sana y puede educar al dueño.

Responsable:

```text
tanques de conocimiento + catálogo de buenas prácticas + fórmulas
```

## 11. Investigación dialéctica

La investigación SmartPyme no es:

```text
upload → procesamiento → PDF
```

Es:

```text
relato
→ hipótesis
→ evidencia
→ contradicción
→ pregunta
→ nueva evidencia
→ cálculo
→ hallazgo
→ explicación
→ seguimiento
```

El sistema debe repreguntar cuando detecta discrepancias mínimas relevantes.

La respuesta del dueño puede ser dato crítico.

Ejemplo:

```text
SmartPyme: “Este archivo parece stock, pero la columna Cant. también podría ser cantidad vendida. ¿Qué representa?”
Dueño: “Es el stock actual que tengo en el local.”
SmartPyme: “Entonces lo tomo como stock actual y lo confronto contra ventas y compras del período.”
```

## 12. BEM.ai dentro de este modelo

BEM no define el negocio.

BEM limpia la entrada.

Rol:

```text
archivos rotos
→ datos limpios estructurados
→ JSON utilizable por SmartPyme
```

SmartPyme decide:

```text
qué investigar
qué fórmula aplicar
qué patología evaluar
qué evidencia falta
qué pregunta hacer
qué hallazgo emitir
```

El schema no debe ser una jaula rígida impuesta por el sistema.

El schema investigativo se configura desde:

```text
rubro
taxonomía operativa
período
objetivo del diagnóstico
dolor del dueño
hipótesis del dueño
selecciones del dueño
archivos disponibles
```

Estos campos no son opcionales salvo decisión explícita del dueño/producto.

### 14.1 Modo rápido

```text

→ reporte automático
```

Útil para diagnósticos simples y reportes base.

### 14.2 Modo profundo

```text
anamnesis
→ hipótesis
→ evidencia
→ BEM
→ aclaraciones
→ contraste histórico
→ hallazgos
→ reporte clínico-operacional
```

Útil para casos de mayor valor, mayor precio y mayor transformación.

---

## 15. Transformación buscada

La meta final no es que el cliente reciba un PDF.

La meta es que el dueño cambie su modelo mental del negocio.

Antes:

```text
“vendo, trabajo, compro, pago, pero no entiendo dónde se me va la plata.”
```

Después:

```text
“mi problema principal está en margen, rotación y precios atrasados; necesito revisar compras, stock inmovilizado y actualización de lista.”
```

Ese cambio mental es el producto profundo.
