## 🗃️ 2. Pipeline de Ingesta & Manejo de Legado (2005-2025)
Los PDFs escaneados antiguos requieren OCR robusto y preservación de metadatos estructurales.

| Etapa | Herramienta | Estrategia |
|-------|-------------|------------|
| **Parsing/OCR** | `Docling` (nativo) + `unstructured` con `tesseract`/`paddleocr` (escaneos) | Fallback automático si `Docling` no extrae texto legible. Preprocesamiento: deskew, binarization, noise removal. |
| **Extracción de Metadatos** | Regex + LLM ligero (`gpt-4o-mini` o local) | `date`, `doc_type` (factura/contrato/acta/excel), `supplier`, `amount`, `page`. Almacenados en JSON inmutable por chunk. |
| **Chunking** | Híbrido: `RecursiveCharacterTextSplitter` (512t, 15% overlap) + `TableChunker` | Tablas → filas con headers preservados. Texto → cortes por párrafo/encabezado. Cada chunk lleva `{"file_hash", "filename", "page", "doc_type", "date", "chunk_id"}`. |
| **Embeddings** | `text-embedding-3-small` (cloud) o `nomic-embed-text` (local) | Costo por ingesta 8.2k docs: ~$3-5 (único). Cache de embeddings en `chroma.sqlite`. |

## 🛡️ 3. Prevención de Alucinaciones & Grounded Prompting
La IA **nunca** debe inventar. Implementamos un triple control:

1. **System Prompt Estricto:**
   ```text
   Eres un consultor de auditoría empresarial. Responde ÚNICAMENTE con la información proporcionada en el contexto recuperado.
   - Si falta info, responde EXACTAMENTE: "No tengo información suficiente en los registros históricos para responder esta consulta."
   - Cita siempre la fuente al final de cada afirmación: [nombre_archivo.pdf, pág. X]
   - No infieras, no resumas sin cita, no uses conocimiento externo.
   - Si el contexto contiene tablas, respeta los valores numéricos originales.
   ```

2. **Validación Post-Retrieve:**
   - Parser de citas: verifica que cada oración con afirmación fáctica tenga `[archivo.pdf, pág. N]`. Si no, rechaza y re-pide o fallback.
   - Threshold de relevancia: `cosine > 0.65` + filtro por metadata (ej. `date >= 2012-07-01 AND date <= 2012-09-30`).

3. **Fallback Explícito:** Si el LLM genera texto sin citas, el pipeline lo intercepta y devuelve el mensaje de "No tengo información...".

## 🔍 4. Arquitectura de Consulta (2s Target & Portabilidad)
### Flujo de Recuperación
```
Query → Metadata Filter (fecha/tipo) → Hybrid Search (BM25 + Vector) → Top-5 → Reranker (local bge-reranker) → Context → LLM → Grounded Output → Streamlit
```

- **BM25 + Vector:** LangChain `ParentDocumentRetriever` o `SelfQueryRetriever`. Mejora recall en términos financieros y fechas.
- **Cache:** SQLite/Redis para queries idénticas. Tasa de acierto esperada >40% en uso contable/operativo.
- **Latencia:** <50ms (filtrado) + <150ms (Chroma) + <300ms (LLM streaming) = **<2s perceptual**.

## 🔎 6. Auditoría de Dinero & Agentic Architecture (Desafío)
**Aclaración técnica:** *"Hermes Agent IA"* no es un framework estándar de la industria. Para auditoría financiera fiable, recomiendo un **Router Agéntico con LangGraph/CrewAI**, no un monolito RAG.

### Diseño Propuesto: `Dual-Track Retrieval`
```
Query → Router (LLM ligero) 
  ├─→ [Track 1: Contextual] → RAG Agent (actas, contratos, notas)
  └─→ [Track 2: Financiero] → Structured Pipeline
        ├─ Parser: Docling + LLM structured output → SQLite (facturas, fechas, montos, PO)
        ├─ Auditor: Pandas/SQL (desfasajes, duplicados, deudas pendientes, variación trimestral)
        └─ Validator: Reglas contables (ej. monto > 0, fecha en rango, moneda consistente)
→ Synthesizer → Grounded Prompt → UI
```

**Herramientas maduras:**
- `LangGraph`: Para orquestación de agentes con estado, fallback y retry.
- `DSPy`: Optimización automática de prompts/routing basada en métricas de precisión.
- `SQLite + Pandas`: Consultas deterministas. **Nunca** confíes sumas o restas al LLM puro.

**Ejemplo de flujo real:**
> *"¿A cuánto le compramos al proveedor X en Q3 2012 y qué decía el acta sobre ese contrato?"*
1. Router detecta `supplier: X`, `date_range: 2012-07-01 to 2012-09-30`
2. Track 2 → `SELECT SUM(amount) FROM invoices WHERE supplier='X' AND date BETWEEN ...`
3. Track 1 → RAG recupera acta de directorio 2012-08-15
4. Synthesizer combina: `"Se adquirió $42,850.00 (ver facturas #1120, #1121, pág. 3, 7). El acta del 15/08/2012 aprobó la renovación del contrato por 2 años adicionales [acta_directorio_2012.pdf, pág. 2]."`

## 2. Decisiones de ingeniería críticas

**Por qué Docling sobre PyMuPDF o pdfplumber.** Los 8.200 archivos incluyen PDFs escaneados desde 2005 que contienen tablas de licitaciones, balances y contratos tabulares. PyMuPDF extrae texto plano y destruye la estructura semántica de las tablas — convierte `| Proveedor X | $45.000 | 2012 |` en `"Proveedor X 45000 2012"`, perdiendo relaciones. Docling mantiene la estructura como Markdown/HTML semántico que el embedder puede procesar con coherencia. Para Excels, el parser debe preservar el nombre de la hoja (`Ventas Q3`) y los encabezados de columna como contexto explícito del chunk.

**Tamaño de chunk y overlap.** Para documentos legales y contables, el tamaño óptimo es `chunk_size=800` tokens con `chunk_overlap=150`. Chunks más pequeños pierden contexto de cláusulas; más grandes diluyen la señal semántica. El overlap es crucial para que cláusulas que cruzan páginas queden recuperables.

**Metadatos como ciudadanos de primera clase.** Cada chunk debe almacenarse en ChromaDB con los siguientes metadatos filtrables: `{"source_file": "contrato_proveedor_x_2012.pdf", "doc_type": "contrato", "date_extracted": "2012-03-15", "page": 4, "sheet": null}`. Esto permite hacer `retriever.filter({"doc_type": "acta", "year": "2012"})` antes del similarity search, reduciendo el espacio de búsqueda dramáticamente.

## 3. Implementación del Grounded Prompt (anti-alucinaciones)

El prompt template es la pieza más crítica del sistema. Debe ser un contrato explícito con el LLM:

```python
GROUNDED_SYSTEM_PROMPT = """
Sos un consultor de registros empresariales. Tu única fuente de verdad son 
los FRAGMENTOS DE CONTEXTO que se te proporcionan a continuación.

REGLAS ABSOLUTAS:
1. Solo respondé con información que figure EXPLÍCITAMENTE en los fragmentos.
2. Si la respuesta no está en los fragmentos, respondé exactamente:
   "No encontré información sobre esto en los registros de la empresa."
3. Cada afirmación debe ir seguida de su cita: [Archivo: nombre_archivo.pdf, Pág. N]
4. No infieras, no supongas, no completes con conocimiento externo.
5. Si los fragmentos son parciales o contradictorios, indicalo explícitamente.

CONTEXTO RECUPERADO:
{context}
"""
```

El `{context}` debe incluir los metadatos del chunk junto al texto: `[Fuente: acta_directorio_2012.pdf | Página 4 | Tipo: acta]\n{chunk_text}`. Sin esto, el LLM cita bien semánticamente pero el usuario no puede verificar el origen.

## 4. Desafío de optimización: el Agente Auditor

Esta es la diferencia entre un buscador avanzado y una herramienta de inteligencia operativa real.

**Arquitectura propuesta para el agente de auditoría.** El agente no usa RAG simple sino un `LangChain AgentExecutor` con herramientas especializadas. Sus `tools` son: `buscar_facturas_proveedor(nombre, fecha_inicio, fecha_fin)`, `comparar_montos_contrato_vs_pago(contrato_id)`, `detectar_gaps_temporales(proveedor)`, y `cruzar_acta_con_operacion(fecha_acta)`. Cada tool ejecuta una cadena de retrieval filtrada sobre ChromaDB y devuelve datos estructurados al agente.

**Por qué no es Hermes Agent específicamente.** Hermes es una familia de modelos fine-tuneados para function calling agresivo (NousResearch). Para datos contables en español con estructuras propietarias, es más robusto usar un agente LangChain con GPT-4o mini + function calling nativo de OpenAI, que tiene mejor comprensión del español técnico financiero y citas más precisas. Si el cliente exige privacidad total (datos sensibles), la alternativa sería `llama-3.1-70b` vía Ollama, que tiene capacidades de function calling aceptables sin salir de la infraestructura local.

**Consulta de ejemplo que el agente puede responder:** "¿A cuánto le compramos al proveedor X en Q3 2012 y qué decía el acta de directorio sobre ese contrato?" requiere que el agente ejecute en secuencia: (1) buscar facturas de proveedor X en el período, (2) buscar contratos con ese proveedor, (3) buscar actas de directorio de 2012 que mencionen al proveedor, y (4) sintetizar con cruce de montos. Eso es una cadena de 3-4 tool calls, perfectamente manejable con `max_iterations=6`.

## 5. Análisis de costos operativos

Con un volumen de 200 consultas diarias y corpus de 8.200 documentos:

**Embedding inicial (one-time):** 8.200 archivos × promedio 15 chunks × 300 tokens/chunk = ~36M tokens. Con `text-embedding-3-small` a $0.02/1M tokens → **costo de indexación total: ~$0.72**. Esto se paga una sola vez.

**Operación mensual:** Cada consulta consume ~2.000 tokens de input (contexto recuperado) + ~500 de output en GPT-4o mini. Con 200 consultas/día × 30 días = 6.000 consultas. 6.000 × 2.500 tokens × $0.60/1M (GPT-4o mini input) + output tokens → **costo LLM mensual: ~$10-12**. ChromaDB local: $0. API Key OpenAI: ya contemplada. **Total mensual estimado: ~$12**, bien por debajo del umbral de $15.

**Opción 100% local con Ollama:** Usando `nomic-embed-text` para embeddings y `llama3.1:8b` para generación, el costo operativo cae a $0 en API pero requiere un servidor con GPU (mínimo una RTX 3080 o una instancia cloud `g4dn.xlarge` a ~$6/mes spot). Para industrias con datos muy sensibles (Legal, Salud, Finanzas reguladas), esto es el camino.

## 6. Estrategia de despliegue y portabilidad

El `Dockerfile` debe tener dos stages: uno de builder que instala Docling y dependencias pesadas, y uno de runtime slim. El `docker-compose.yml` levanta el servicio RAG + ChromaDB como volumen persistente montado en `/data/chroma`. La variable de entorno `DEPLOYMENT_MODE=local|cloud` controla si ChromaDB usa persistencia en disco o el cliente HTTP a una instancia gestionada.

Para producción en Azure, `Azure Container Apps` es la elección correcta: escala a cero cuando no hay consultas (costo cero en horas off-peak), tiene integración nativa con Azure Blob Storage para los archivos originales (requerimiento para el "link al PDF original" en Streamlit), y cumple con los estándares de compliance que cualquier PYME con 20 años de historia va a necesitar al crecer.

El pipeline de ingesta debe ser idempotente: cada archivo tiene un hash SHA-256 como `doc_id` en ChromaDB. Si se re-ejecuta el ingester sobre el mismo archivo, detecta el ID existente y lo saltea. Esto permite agregar documentos nuevos al corpus sin re-indexar los 8.200 archivos existentes.
