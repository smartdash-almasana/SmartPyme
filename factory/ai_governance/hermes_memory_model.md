# Hermes Memory Model

## Capas

- perceptiva: ciclo actual (repo, git, hallazgo)
- corta: datos del ciclo → evidence
- larga: skills, specs, contratos versionados

## Reglas

- no mezclar memoria vieja con percepcion
- cada ciclo arranca limpio
- evidencia persiste, memoria no

## Aprendizaje

solo via repo:

- evidencia -> hallazgo mejora -> validacion -> skill/spec

## Principio

Hermes aprende rutinas, no contexto conversacional.
